# itamatrix — שפה טבעית ← חיפוש ב-ITA Matrix

תרגום עברי של [SKILL.md](SKILL.md). מסמך ה-skill הראשי (כולל בלוק הפקודות
הנוצר אוטומטית) נשאר באנגלית; כאן התקציר וההסבר בעברית.

ה-skill מפעיל את ה-CLI בשם `itamatrix`. התפקיד: להפוך בקשת טיסה בשפה חופשית
לפקודה ולדגלים הנכונים, ולקודד כוונות מתקדמות כ**קודי ניתוב** (routing — המסלול)
ו**קודי הרחבה** (extension — תמחור/סינון) של ITA.

## דרישות מוקדמות

ה-CLI מותקן כ-`itamatrix` (או `npx itamatrix ...`). בהרצה ראשונה ייתכן שיבקש
דפדפן: `npx playwright install chromium`. שאילתות עלולות להימשך זמן רב (Matrix
איטי בצד השרת); שאילתות חוזרות נשענות על המטמון. **קבעו timeout של לפחות 180 שניות**
לפקודה — חיפוש עשוי לרוץ כך עד שיחזיר. אל תפסיקו מוקדם ותסיקו שנכשל.

הריצו תמיד עם `--json` כדי לקבל פלט מובנה לניתוח וסיכום.

## בחירת הפקודה

| צורת הבקשה | פקודה |
|------------|-------|
| כיוון אחד / הלוך-ושוב A→B | `search <origin> <dest> --depart … [--return …]` |
| 3+ רגליים נפרדות / open-jaw | `multicity --leg O:D:DATE --leg …` (≥2 רגליים) |
| "התאריך הזול לטוס", תאריכים גמישים | `calendar <origin> <dest> --depart-range S:E [--trip-length N]` |

הקודים הם קודי שדה-תעופה/עיר (BOS, LON). תאריכים בפורמט `YYYY-MM-DD`. תאריך יחסי
("חודש הבא") — המירו לתאריך מוחלט תחילה.

## מיפוי כוונה ← דגלים

העדיפו דגלים פשוטים לכוונה פשוטה; פנו ל-`--routing`/`--ext` רק כשדגל לא מספיק.

| כוונה | שימוש |
|-------|-------|
| מחלקה | `--cabin cheapest\|premium-economy\|business\|first` |
| ישיר / מקס' עצירות | `--stops none\|1\|2` |
| "רק UA ו-AA" (פשוט) | `--carriers UA,AA` |
| מספר נוסעים | `--adults N` |
| כמה תוצאות | `--limit N` (1–25 ל-search/multicity) |
| בניית תעריף + קישור Google Flights | `--details` (ל-search/multicity) |

## קודי ניתוב / הרחבה לכוונות מתקדמות

קראו את **[../../docs/ROUTING_CODES.md](../../docs/ROUTING_CODES.md)** לדקדוק המלא
לפני בניית קודים. כללי אצבע:

- **כוונת מסלול ← `--routing`.** אילו חברות/שדות ובאיזה סדר. לדוגמה: "דרך טוקיו או
  סיאול" → `--routing NRT,ICN`; "לכפות קונקשן בדאלאס" → `--routing X:DFW`.
- **כוונת סינון/תעריף ← `--ext`.** מגבלות עצירה, משכים, ברית, ללא טיסות לילה, בסיס
  תעריף. הפרידו פקודות ב-`;`. לדוגמה: "עצירות קצרות, ללא לינה" →
  `--ext "MINCONNECT 0:45; MAXCONNECT 2:00; -OVERNIGHTS"`.
- אל תערבבו: מסלול ב-`--routing`, סינון/תעריף ב-`--ext`.

## בניית תעריף וקישור Google Flights (`--details`)

`--details` (ב-`search` וב-`multicity`) פותח את **עמוד הפירוט של התוצאה הזולה**
ומוסיף שני דברים לפלט:

- **Fare Construction** — פירוק ה-NUC של בסיס התעריף, ש-ITA מתייג כ"שימושי לסוכני
  נסיעות" (למשל `BOS B6 LON 70.00OL8LBVL1 NUC 70.00 END ROE 1.00 XT …`). הציגו
  כפי שהוא; סוכנים/דלפקי הוצאת כרטיסים קוראים אותו ישירות.
- **Open in Google Flights** — קישור עומק של עמוד הפירוט
  (`google.com/travel/flights?tfs=…&source=ita_matrix`) לאותו מסלול.

זה עולה ניווט נוסף ו**עוקף את המטמון** (הקישור דורש את ה-session החי), אז העבירו
רק כשהמשתמש רוצה את פירוט התעריף או קישור להעברה. הפירוט הוא best-effort: אם העמוד
לא נפתח — תוצאות החיפוש עדיין חוזרות, רק בלי בלוק ה-`details`. ב-JSON זהו שדה
עליון `details: { fareConstruction, googleFlightsUrl }`; בפלט טבלה זהו footer מתחת
לתוצאות. **הציגו את שניהם למשתמש כשהם קיימים.**

## תהליך עבודה

1. נתחו את הבקשה: מסלול(ים), תאריכים, נוסעים, ואילוצים.
2. בחרו פקודה; המירו תאריכים יחסיים למוחלטים.
3. מפו אילוצים לדגלים תחילה; קודדו את השאר כ-`--routing` / `--ext`.
4. **הציגו את הפקודה הבנויה למשתמש לפני ההרצה** — במיוחד קודי הניתוב/הרחבה.
5. הריצו עם `--json`, נתחו, והציגו תוצאות בבירור (מחיר, חברות, עצירות, משך). על
   שגיאת CLI — קראו את ההודעה (היא מציינת את הדגל הבעייתי) ותקנו, לא לנחש.

## דוגמאות

> "הטיסה הזולה במחלקת עסקים מבוסטון ללונדון באוגוסט הבא, oneworld בלבד, ישירה."

```bash
itamatrix --json search BOS LON --depart 2026-08-15 \
  --cabin business --stops none --ext "ALLIANCE oneworld"
```

> "הכי זול מבוסטון ללונדון, ותן לי את בניית התעריף וקישור Google Flights לזולה."

```bash
itamatrix --json search BOS LON --depart 2026-08-15 --details
# → result.details.fareConstruction[] + result.details.googleFlightsUrl
```
